Retro Skill

Sprint retrospective with cross-log discovery and sprint closure

The sprint lifecycle is planning → dev → retro → next cycle. Planning and dev are implemented; retro is the missing piece that closes the loop. Per Decision 34, retro owns sprint closure.

Acceptance Criteria 9 ACs

  1. A skill exists at skills/momentum/skills/retro/SKILL.md invocable via /momentum:retro.
  2. The skill reads all agent JSONL log files from .claude/momentum/sprint-logs/{sprint-slug}/ and correlates events to build a holistic timeline.
  3. Verifies every story reached done; otherwise flags closed-incomplete.

Cross-Story Pattern Detection

Surface recurring shapes across stories so practice improves over time

The flywheel hinges on noticing that the same kind of mistake or ambiguity has shown up before. This story builds the detection layer that reads findings-ledger.jsonl and emits cluster signals to the next sprint's planning surface.

Acceptance Criteria 6 ACs

  1. Reads findings-ledger.jsonl across the last N sprints (default 5).
  2. Clusters findings by category + tokenized summary; minimum cluster size 2.
  3. Emits cross-sprint-patterns.md per planning cycle.

A 4-point warmth shift. Subtle but present in peripheral vision. Combined with the type bump (17→18px) and serif ACs, the surface announces itself as a doc.
strip B · no flip Light → reading is a property change, not a polarity flip. No cross-fade — that would over-signal. Instead the paper warms and the type tunes; the change is felt rather than announced.
why warm, not cool Warmer paper (#faf6ec) reads as physical — closer to vellum than to a screen. Reinforces "this is a document." Cool would have pushed the opposite direction.
assumption flagged One toggle, two effects: the rail switch only flips dashboard polarity. Reading stays reading. If users want to read in dark they can't — a future tweak might add a per-surface override, but I'd resist it: the polarity rule is the design decision, not a preference.
back-navigation Going back (L3 → L2 → dashboard) reverses the same rule. From dark origin: 140ms cross-fade returning to dark. From light origin: paper cools back to #fbfaf7, no fade. Not shown to keep this canvas focused on entry.
