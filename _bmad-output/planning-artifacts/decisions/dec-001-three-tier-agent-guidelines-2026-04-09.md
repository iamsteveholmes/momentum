# DEC-001: Three-Tier Agent Guidelines Architecture

## Decision

Momentum's agent guidelines system uses a **three-tier architecture**. The
`momentum:build-guidelines` skill (gen-2 replacement for `momentum:agent-guidelines`)
generates Tier 1 and Tier 2 artifacts. Tier 3 is maintained by `momentum:kb-init` and
`momentum:kb-ingest`.

**Guidelines are NOT injected at runtime via spawn prompts.** They are baked into
composed specialist agent files before the sprint starts.

---

## Architecture

### Tier 1 — Hot Constitution (always loaded)

```
.claude/guidelines/constitution.md
```

~660 lines. Always in context. Contains:
- Trigger tables (domain → reference doc pointer)
- Critical cross-cutting rules (commit format, authority hierarchy)
- Pointers to Tier 3 cold KB for on-demand retrieval

### Tier 2 — Composed Specialist Agent Files (system prompt)

```
.claude/guidelines/agents/{role}-{domain}.md
```

Example: `dev-kotlin-compose.md` = `dev.md` body + Kotlin Compose project manifest
merged into one file. The system prompt IS the guidelines — they are not injected at
spawn time.

Generated by `momentum:build-guidelines`. One file per role×domain combination:
`dev-kotlin-compose.md`, `dev-python-fastapi.md`, `qa-kotlin-compose.md`, etc.

### Tier 3 — Cold Knowledge Base (on-demand, separate repo)

```
{project}_vault/
├── raw/        ← Immutable source material
├── wiki/       ← Pre-synthesized pages (LLM-maintained)
│   ├── concepts/
│   ├── entities/
│   ├── sources/
│   └── maps/
├── index.md    ← Registry
├── log.md      ← Ingest history
└── CLAUDE.md   ← Vault navigation schema
```

Separate git repo per project. Fed by `momentum:kb-ingest`. Agents access via
index-first navigation — grep `index.md`, read targeted page, never load all pages.

Reference example: `~/projects/momentum_vault` (the cold KB for Momentum itself).

---

## Why This Architecture

### Problem: CLAUDE.md inheritance is unreliable

GitHub issue #29423 (open April 2026): Task subagents do NOT reliably load project
CLAUDE.md, `.claude/rules/`, or `~/.claude/CLAUDE.md`. Confirmed by concrete failure
case — 5 missed violations when subagents ran without project rules.

GitHub issue #8395 (closed NOT_PLANNED): Automatic rule propagation to subagents is
not on the Anthropic roadmap.

GitHub issue #16299 (open): Path-scoped rules have a regression — rules load globally
instead of conditionally on file access.

**The only guaranteed delivery channel to a subagent is its system prompt body.**

### Problem: Runtime injection dilutes attention budget

Sonnet-class models show linear guideline adherence decay from the first instruction.
Reasoning models hold through ~100-150 instructions then cliff. Adding guidelines to
the spawn prompt (what the agent is "asked to do this time") competes with task
instructions for the same attention budget.

System prompt content (who the agent IS) does not compress the same way as spawn
prompt content. It sits at the top of context where attention is strongest.

### Why not RAG?

Karpathy's argument (applied in the April 9 session): "RAG doesn't accumulate. Every
query re-derives knowledge from scratch. The alternative is a persistent wiki where
decisions compound over time." The cold KB is a maintenance and storage pattern — agents
retrieve targeted slices from it, not the whole thing.

### Why three tiers, not two?

Production reference architecture (Codified Context study, 2026): systems that work at
scale universally split knowledge into hot (always-loaded) and cold (on-demand) tiers.
The specialist agent tier sits between them — domain knowledge baked into the agent
definition body at preparation time, not loaded at query time and not retrieved on demand.

---

## What Changed from Gen-1

Gen-1 (`momentum:agent-guidelines`, status: done) generates:
- `.claude/rules/*.md` — path-scoped constraint files ✓
- `docs/references/*.md` or `docs/guidelines/*.md` — wiki chunks ✓

Gen-1 does NOT generate:
- `constitution.md` — the hot always-loaded layer
- Composed agent files in `.claude/guidelines/agents/` — the baked-in system prompts
- A separate cold KB vault repo

Gen-2 (`momentum:build-guidelines`) generates all three Tier 1 and Tier 2 artifacts,
distilling from the Tier 3 cold KB maintained by `kb-init` + `kb-ingest`.
