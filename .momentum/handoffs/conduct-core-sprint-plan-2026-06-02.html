00What this sprint is — and the DEC-036 through-line

Conduct collapses sprint-dev's multi-gate firehose into an autonomous per-story build with one human gate at the end (DEC-035). DEC-036 then found conduct legible but stakes-blind and amended DEC-035's binding decision #1: a narrow, high-bar, stakes-gated mid-flight escalation tier is now permitted, stakes-class findings leave the silent auto-fix path, the report renders dismissals, and the end-gate gains an anti-rubber-stamp forcing function. This slice builds that core — the escalation engine, the finding schema + shared stakes rubric, the Conductor spine, the bmad-code-review adapter, the directed-dev fix leg, and the dev/qa rescope.

The load-bearing rule — quote it when a judgment call gets fuzzy Don't remove HITL — remove the safely-handled decisions so the human isn't overwhelmed, while raising safety and review ability. Gate the human only for stakes classes (security/auth-isolation; irreversible/destructive; high-blast-radius/architecture), routed by what (stakes) and when (timing). The mid-flight bar stays narrow — irreversible-and-imminent OR build-invalidating only — or it re-becomes the firehose DEC-036 exists to kill. Routine findings stay always auto-fixed.

Governed by DEC-035 (adopt conduct; one end-gate; no story-count cap; functionality-organized report; legible auto-fix) and DEC-036 (stakes-and-timing escalation; render dismissals; anti-rubber-stamp; decision-grade presentation). The 2–8 story sprint cap is intentionally overridden — this is a ~21-story feature-grained slice by design (testing DEC-035's no-cap model). Built in a markdown/bash repo, so every story is a skill, agent-definition, or spec doc — all built by momentum:dev-skills; there is no code/UI lane.
