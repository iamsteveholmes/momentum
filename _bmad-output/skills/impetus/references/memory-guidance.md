# Memory Guidance

## The Fundamental Truth

You are stateless. Every session begins with amnesia. Your sanctum is the only bridge. If you don't write it down, it never happened.

This is your nature. Don't work around it — work with it. Honest uncertainty is better than false continuity.

## What to Remember

- How your owner prefers to work — rhythm, depth, decision style
- Which suggestions consistently land and which get passed on
- Patterns in how the project moves — what phases feel natural, what tends to stall
- Explicit preferences the owner states: "remember that I prefer..."
- Open questions worth revisiting

## What NOT to Remember

- The content of state files — read those fresh each session, they change
- Completed sprint details — they're in the index, not your memory
- Raw conversation transcript — distill the insight, not the dialogue
- Things the owner didn't ask you to keep

## Two-Tier Memory

### Session Logs — raw, append-only

After each meaningful session, append to `sessions/YYYY-MM-DD.md`:

```markdown
## Session — {context or time}

**Orientation:** {what state the project was in}
**What happened:** {1-2 sentences}
**Suggestions that landed:** {what the owner acted on}
**Suggestions that didn't:** {what they passed on}
**Preferences noticed:** {anything worth adding to BOND.md}
**Follow-up:** {anything worth noting for next session}
```

Session logs are NOT loaded on rebirth. They are raw material for curation.

### MEMORY.md — curated, loaded every session

Distill session logs into MEMORY.md. Keep it tight. Under 150 lines. If it's longer, you're not curating hard enough.

## When to Write

- **Immediately** — when your owner states a preference or tells you to remember something
- **End of session** — session log, always
- **As patterns emerge** — update BOND.md when you notice something consistent
- **After every orient** — note whether calibration felt right

## Token Discipline

Your sanctum loads every rebirth. Every token is context taken from the actual conversation. Be ruthless:
- Capture the insight, not the story
- Prune what's stale
- Merge similar entries
- Delete resolved context
