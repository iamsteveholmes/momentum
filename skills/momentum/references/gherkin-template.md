# Gherkin Spec Template

All `.feature` files generated by Momentum must follow this template.

## Format

```gherkin
Feature: {Story title}

  Background:
    Given {common precondition shared by all scenarios}

  Scenario: {Short behavioral description — what the system does}
    Given {observable starting state}
    When {user action or system event}
    Then {observable outcome}
    And {additional observable outcome if needed}
```

## Voice and Tense

- **Given:** Past participle or present state — "the developer is on the main branch", "a story file exists"
- **When:** Present tense, active voice — "the developer invokes", "the validation team runs"
- **Then:** Present tense, observable — "a warning appears", "the story is registered", "the merge completes"
- **Subject:** Use "the developer" for user actions, "the system" or the specific agent name for system behavior
- **No passive voice in When:** "When the developer invokes" not "When quick-fix is invoked"

## Naming

- Scenario names describe **what the system does**, not which requirement is tested
- Good: "Developer reviews story in rendered surface before proceeding"
- Bad: "AC 2 — Phase 1 creates story and opens surface"
- No AC numbers, no phase numbers in scenario names
- Keep scenario names under 80 characters

## Structure Rules

- One Background per feature — shared preconditions only
- Background should be minimal — 1-2 Given lines maximum
- Each scenario is self-contained with its own Given/When/Then
- No And in When — split into separate When if multiple actions
- And is fine in Given and Then for multiple conditions/outcomes
- No But — rephrase as positive Then

## Indentation

- Feature: no indent
- Background/Scenario: 2 spaces
- Given/When/Then: 4 spaces
- And: 4 spaces (same level as Given/When/Then)

## What NOT to Include

- No tags (@wip, @skip, etc.) — Momentum doesn't use a Gherkin runner
- No Scenario Outline/Examples — keep scenarios concrete
- No comments (`#`) — scenario names should be self-explanatory
- No data tables unless the scenario genuinely requires structured input
