## Story

As a sprint-dev orchestrator,
I want cache coverage analysis results by target and never re-spawn coverage agents for already-analyzed targets,
so that 126 wasted turns per sprint in nornspun are eliminated — coverage analysis is idempotent across the sprint lifecycle.

## Description

Coverage analysis agents are re-spawned for the same targets across phases. Each re-spawn burns 10-15 turns analyzing files that haven't changed. A cache keyed by (target-slug, file-hash) would make coverage analysis idempotent within a sprint.

**Pain context:** Nornspun upstream #2 (Critical). 126 wasted turns per sprint observed. Coverage agents are one of the top token consumers in sprint-dev.
