## Story

As a dev/fix agent,
I want spawn a read-only investigation agent that maps the problem space before any fix agent is granted write access,
so that fixes are grounded in full problem understanding — agents don't jump to solutions before understanding what's connected.

## Description

Fix agents are spawned with write access before the problem is fully understood. This leads to partial fixes, wrong assumptions, and git contention when multiple agents write simultaneously. A read-only investigator spawned first produces a findings brief; the fix agent receives that brief and has a clear, bounded change set.

**Pain context:** Nornspun upstream #3 (High). Git contention during parallel fix attempts. Agents writing files before completing investigation. Investigation findings not shared between agents.
