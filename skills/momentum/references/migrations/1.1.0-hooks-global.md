# 1.1.0 — Update hooks to global path with project-local override

Read `${CLAUDE_SKILL_DIR}/references/hooks-config.json`.
Read existing `.claude/settings.json` (start with `{}` if absent).

For each hook event key in hooks-config (`PostToolUse`, `PreToolUse`, `Stop`):
- Find existing Momentum hook entries (those whose `command` contains `.claude/momentum/hooks/`)
- Replace them with the updated commands from hooks-config.json (which now use the global path with project-local override)
- If no Momentum entries exist, add them as new entries

The updated commands use the pattern:
```
bash -c 'LOCAL="$CLAUDE_PROJECT_DIR/.claude/momentum/hooks/<script>"; GLOBAL="$HOME/.claude/momentum/hooks/<script>"; if [ -f "$LOCAL" ]; then exec bash "$LOCAL"; elif [ -f "$GLOBAL" ]; then exec bash "$GLOBAL"; fi'
```

This ensures:
1. Project-local scripts (`.claude/momentum/hooks/`) take precedence if they exist
2. Falls back to globally installed scripts (`~/.claude/momentum/hooks/`)
3. If neither exists, the hook exits silently (no error) — safe for non-Momentum projects

Write merged result to `.claude/settings.json` — preserve ALL existing keys.

Note: Existing project-local copies in `.claude/momentum/hooks/` continue to work during
transition — they take precedence via the override pattern above.
