## Story

As a developer,
I want an sm.md base role body that runs sprint-planning, create-story, and retro workflows with lightweight technical literacy and a hard escalation limit,
so that the SM role has a well-scoped base body for composition into project-specific specialist agents.

## Description

Create sm.md as a base role body that runs sprint-planning, create-story, and retro workflows. Lightweight technical literacy (story-quality judgment, change-type literacy, touches-path shape-knowledge) without deep technical depth. Hard limit: escalate implementation design questions to developer or specialist dev. Depends on SM technical proficiency scope decision.

**Pain context:** The SM role currently has no base body in the plugin. Without sm.md, the scrum master role cannot be composed per-project via build-guidelines. The SM's lightweight technical literacy boundary (vs. deep technical depth) is a design decision that must be captured in the base body to prevent scope creep in composed specialists.
