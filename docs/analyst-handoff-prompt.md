### Key Design Decisions Already Made
- **Dev agent is the universal spec executor** — both product stories and process tasks flow through the dev agent. Process task specs tell the dev agent to invoke BMB agents.
- **Quick Spec is the specification mechanism** for process tasks — lighter than full stories, but still has acceptance criteria.
- **Global vs project scope** — practice rules go in `~/.claude/` (deployed from Momentum canonical source). Project-specific config goes in `./.claude/` (committed to project repo, generated by bootstrap).
- **Momentum is source of truth** — `~/.claude/` is deployed state. `momentum install` deploys it. New machine = clone any project with Momentum, run install.
- **Project boundary = coding standard boundary** — dissimilar stacks get separate projects.
- **No ATDD for process tasks** — config/markdown files are verified by review and mechanical checks, not automated test suites.
- **The Momentum module may be temporary** — BMAD is undergoing major changes (skills-based architecture, workflow customization/replacement). Design for adaptability.
