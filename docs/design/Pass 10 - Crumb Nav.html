Cross-Story Pattern Detection

Surface recurring shapes across stories so practice improves over time

The flywheel hinges on noticing that the same kind of mistake or ambiguity has shown up before. This story builds the detection layer that reads findings-ledger.jsonl and emits cluster signals to the next sprint's planning surface.

Per Decision 41, clustering runs on demand inside retro Phase 5 (distill) — not as a background job. The output lives at cross-sprint-patterns.md and is read by /momentum:plan at the next planning step.

Acceptance Criteria 6 ACs

  1. Reads findings-ledger.jsonl across the last N sprints (default 5).
  2. Clusters findings by category + tokenized summary; minimum cluster size 2.
  3. Emits cross-sprint-patterns.md per planning cycle.
  4. Surfaces top 3 clusters in the planning workflow's intake.
  5. Records cluster IDs back to findings-ledger.jsonl for triage tracking.
