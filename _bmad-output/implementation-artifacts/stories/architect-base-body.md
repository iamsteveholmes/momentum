## Story

As a developer,
I want an architect.md base role body that knows architecture.md, owns architectural-impact judgment, and runs architecture-guard drift-detection patterns,
so that the architect role has a well-scoped base body for composition into project-specific specialist agents.

## Description

Create architect.md as a base role body that knows architecture.md, owns architectural-impact judgment during sprint-planning, and runs architecture-guard drift-detection patterns. Bounded scope: architecture document updates and architecture impact analysis, not code design.

**Pain context:** The architect role currently has no base body in the plugin. Without architect.md, any project needing architecture review during sprint-planning cannot compose a project-specific architect specialist via build-guidelines. The role exists in the practice conceptually but has no canonical base body.
