## Story

As a AVFL validator,
I want complete both enumeration and adversarial challenge in a single agent pass before returning findings,
so that coverage validation is self-challenging — the agent finds its own blind spots without a separate adversary spawn.

## Description

Current AVFL uses separate Enumerator and Adversary agents. For coverage validation specifically, a single agent doing both phases (enumerate all items, then challenge whether the enumeration is complete) may produce equivalent quality with half the spawns. This is a design spike to evaluate.

**Pain context:** Nornspun upstream #12 (Low). Separate spawn overhead for enumerator/adversary is significant at scale. If two-phase single-agent produces equivalent adversarial pressure, it's a pure optimization.
