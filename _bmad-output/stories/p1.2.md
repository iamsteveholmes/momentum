## Story

As a Momentum developer,
I want `momentum-plan-audit` to run automatically before plan mode exits,
So that I never need to remember to invoke the skill manually — the audit just happens.

## Background

Story p1.1 built the `momentum-plan-audit` skill and a `PreToolUse` hook on `ExitPlanMode`. The hook can only run bash — it can't invoke an LLM skill. So the UX was: try to exit → get blocked → manually run the skill → try again. That's the wrong flow.

The correct mechanism is a Claude rule (`.claude/rules/`). Rules are LLM instructions that auto-load every session. The rule checks for `## Spec Impact` before `ExitPlanMode` and invokes `momentum-plan-audit` automatically if the section is absent. The hook becomes a safety net.

## Acceptance Criteria

**Given** the user is in plan mode and calls `ExitPlanMode`,
**When** the active plan file does not have a `## Spec Impact` section,
**Then** Claude automatically invokes `momentum-plan-audit` before proceeding — without the user having to run it manually.

---

**Given** the active plan already has a `## Spec Impact` section,
**When** the user calls `ExitPlanMode`,
**Then** Claude proceeds directly without invoking `momentum-plan-audit`.

---

**Given** the `PreToolUse` hook fires on `ExitPlanMode`,
**When** `## Spec Impact` is absent (rule failed to run),
**Then** the hook blocks with a safety-net message explaining the rule should have handled this.

## Definition of Done

- [ ] `.claude/rules/plan-audit.md` — rule instructs Claude to invoke `momentum-plan-audit` if `## Spec Impact` absent before `ExitPlanMode`
- [ ] `check-plan-audited.sh` — block message updated to identify itself as a safety net
- [ ] Hook unit tests still pass (exit codes unchanged)
