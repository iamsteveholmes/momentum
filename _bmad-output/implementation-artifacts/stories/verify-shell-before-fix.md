## Story

As a dev/fix agent,
I want verify the target shell type (zsh, bash, sh) before suggesting or executing shell-specific commands,
so that shell compatibility errors are eliminated — commands work in the developer's actual shell, not just the agent's assumed one.

## Description

Agents suggest commands using bash-specific syntax in zsh environments (or vice versa), causing failures. A pre-fix shell check reads the shell from the environment or cmux session context before generating any shell commands.

**Pain context:** Nornspun upstream #10 (Medium). Shell mismatch errors were observed in fix agents. The fix was correct logically but syntactically wrong for the target shell.
