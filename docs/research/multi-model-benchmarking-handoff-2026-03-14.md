## Key Decisions Already Made (from research)

1. **Primary eval tool**: Promptfoo (OSS, MIT, pending OpenAI acquisition). Has a Claude Agent SDK provider that can test full agentic workflows, not just single prompts.
2. **Default model strategy**: Sonnet 4.6 at medium effort as the general-purpose default. Opus for orchestration/complex reasoning. Haiku for constrained subtasks.
3. **Quality evaluation**: LLM-as-judge with Opus as grader, using multi-dimensional rubrics (accuracy, completeness, coherence, domain fitness, usability).
4. **Observability**: `claude -p --output-format json` for programmatic benchmarking. OTel via `CLAUDE_CODE_ENABLE_TELEMETRY=1`. ccusage for cross-session tracking.
5. **The cognitive load dimension**: Model selection should consider not just error quantity but error detectability by human reviewers. Flagship models make fewer invisible errors.

## What the Implementation Should Produce

### 1. Promptfoo Configuration for BMAD Skills
A `promptfooconfig.yaml` (or set of configs) that:
- Tests specific BMAD skills (session prep, creature building, etc.) across Opus/Sonnet/Haiku
- Uses the Claude Agent SDK provider to run full skill executions
- Includes `cost` and `latency` assertions
- Uses `llm-rubric` with a multi-dimensional rubric scored by Opus
- Supports `evaluateOptions.repeat: 3` for statistical consistency

### 2. Bash Benchmarking Script
A script that:
- Runs `claude -p --model X --output-format json` for each model
- Captures wall time, API time, cost, token counts from JSON output
- Produces a comparison table (model, time, cost, tokens)
- Optionally feeds outputs to an LLM judge for quality scoring

### 3. Golden Dataset (Starter)
- 5-10 reference outputs per skill type, generated by Opus and human-reviewed
- Stored in a format promptfoo can consume as test cases

### 4. Pydantic AI Benchmarking Harness (if applicable)
- Python script using `agent.override(model=X)` + Pydantic Evals
- Captures `result.usage()` for token/cost tracking
- Runs the same agent across N models and produces comparison output

### 5. Model Configuration for Existing Skills
- Review current BMAD skills and sub-agents
- Add `model:` frontmatter to skills where a non-default model is appropriate
- Set up `CLAUDE_CODE_SUBAGENT_MODEL` or per-agent frontmatter for cost optimization

## Risks and Caveats

- **Promptfoo OpenAI acquisition**: Announced 2026-03-09, pending close. Promptfoo committed to MIT and multi-provider support. Monitor for neutrality issues when evaluating Claude models. Langfuse and DeepEval are provider-neutral alternatives.
- **Benchmark numbers**: Some 4.6-era benchmarks (GPQA Diamond, HLE without-tools, Terminal-bench gap) could not be verified from extractable text -- they appear in blog post images. The numbers used are from the best available third-party sources and are not contradicted by official sources.
- **Arena rankings are volatile**: Specific rank numbers (Opus #1, Haiku ~#58) shift frequently. Use as directional guidance, not absolutes.
- **Gemini 3.x is in Preview**: The guide now includes Gemini 3.x models but notes they are Preview status. For production stability, Gemini 2.5 stable remains safer.
- **OpenAI models in transition**: GPT-4o, GPT-4.1, o4-mini retired from ChatGPT (Feb 2026), replaced by GPT-5.3/5.4. Legacy models still available via API.
