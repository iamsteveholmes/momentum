The Momentum Cycle

Six phases on a continuous loop. Each sprint ends by writing handoff events to the practice ledger — and the next sprint-planning reads them in its first step. The cycle is a flywheel: every rotation compounds.

    Mid-flight escalation — the only interruption rule

    The pipeline runs uninterrupted unless a finding is stakes-class and either irreversible-and-imminent or build-invalidating. Everything else — however interesting — waits for the single human end-gate.

    The deterministic spine

    Discipline that does not depend on a model remembering to behave. These mechanisms execute automatically — they cannot be skipped, deprioritized, or forgotten.

    Three hooks

    file-protection (PreToolUse) · lint-format (PostToolUse) · stop-gate (Stop). Deterministic enforcement that fires on every matching event, no exceptions.

    Sole-writer rules

    sprint-manager is the only writer of sprints/index.json and stories/index.json; epic-grooming owns epics.json. Enforced by the file-protection hook via protected-paths.json.

    Frozen contracts

    Verification contracts — .feature, .eval.yaml, .smoke.sh, .review.md per change_type — are locked into .momentum/sprints/{sprint}/specs/ at sprint activation.

    Practice ledger

    .momentum/practice-ledger.jsonl — a structured event log of quality findings, triage outcomes, and consumption records across sprints. The next sprint-planning reads it first. This is the flywheel's memory.

    Producer–verifier separation

    Everywhere in the cycle: writers never verify, verifiers never write. Reviewers run read-only; the implementer returns in fix-mode only with directed findings in hand.

    Canonical finding schema

    Every finding carries a stakes_class: security-auth-isolation · irreversible-destructive · high-blast-radius-architecture · routine. Legitimate + routine auto-fixes; legitimate + stakes-class escalates; non-legitimate is dismissed with rationale; out-of-scope is triaged.

    Alongside the wheel

    Quality and infrastructure skills run beside the six phases rather than inside any one of them — validation loops, drift detection, the live dashboard, and the 3-tier agent composition system (hot constitution / composed agent / cold KB).
