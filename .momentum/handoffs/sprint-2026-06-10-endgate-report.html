01What shipped

In one line: the Momentum build engine can now run a sprint without a human babysitting the parts it used to fumble.

Before this sprint, the Conductor — the in-session engine that builds an entire sprint autonomously — had five holes that forced a human (or a lucky guess) to paper over them on every run: it passed a worktree path it never created, it kept all its build state in volatile memory that a crash would erase, it told the developer agent both to commit and not to commit, it consumed a QA finding format nobody had built the converter for, and its “defer QA” option silently deleted the adversarial code review too.

After this sprint, each of those is closed in the skill: the Conductor explicitly creates every story’s branch and worktree from the sprint tip; it journals every build event to an append-only ledger that survives session death and powers resume; it is the single commit owner with a guard that now actually fires; the QA reviewer’s findings are normalized to the shape the fix loop expects; and deferring QA leaves the code review fully intact.

The proof is unusually direct: because the engine ran this build without the persistence feature, it wrote its own journal by hand — and that 139-row ledger on disk is the working demonstration of exactly the behavior the persistence story specifies. The completeness caveat: this was validated by structured inspection and the shipped evals, not by a live re-run of the rebuilt engine, and one story’s path (covered-by-composition) was exercised by spec and evals only. Full honesty in §06.

02What each piece is for

One plain paragraph per story — its job, the guarantee it adds, and what breaks without it. Expand any item for how it was verified (testing first), the rationale, and the diff.

Durable build ledger — the end-gate and resume survive session deathcritical conduct-build-state-persistence-and-resume

Its job. Writes every state-bearing build event to an append-only journal on disk the instant it happens, so the single human end-gate and partial-run resume can be rebuilt even if the session dies or its context is compacted mid-build.

The guarantee. At story 20 of a 25-story build, a crash loses at most the one event in flight — never the findings, dispositions, escalations, or quarantine records already recorded.

What breaks without it. Without it, all of that lives only in volatile in-context memory; a session death silently corrupts the one gate the whole design leans on.

D2 · The build journal is written with a shell command that breaks on an apostrophe SECURITY

In plain terms. The new build-ledger reference tells the Conductor to append each journal row with printf ‘%s\n’ ‘{…json…}’ — the row’s JSON wrapped in shell single-quotes. Journal rows routinely carry free prose (finding summaries, dismissal rationales, developer change-request text).

What’s actually at stake.
